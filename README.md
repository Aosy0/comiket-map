# C107 サークルマップ

コミックマーケット107用のオフライン対応サークルマップ PWA アプリケーション

## 概要

コミックマーケット当日に、オフラインまたは最小限の通信量でサークル情報を確認できるスマートフォン向けWebアプリです。

## 機能

- 📋 **サークル管理** - サークル名、スペース番号、ジャンル、メモを登録・編集・削除
- 🗺️ **マップ表示** - サークルマップ画像をピンチズーム・パン操作で閲覧
- 📅 **日程フィルター** - 1日目/2日目でサークルを絞り込み
- 🔍 **検索機能** - サークル名・スペース番号で検索
- ✅ **チェック機能** - 訪問済みサークルをマーク
- 📤 **エクスポート/インポート** - JSON形式でデータをバックアップ・復元
- 📱 **PWA対応** - ホーム画面に追加してアプリのように使用可能
- 🔌 **オフライン対応** - 一度アクセスすればオフラインで動作

## データ保存について

### 保存されるデータと期間

このアプリは以下のデータをブラウザ内に保存します：

| データ種類 | 保存場所 | 内容 | 保存期間 |
|----------|---------|------|---------|
| **ユーザーデータ** | LocalStorage | 登録したサークル情報、チェック状態 | **半永久的** |
| **アプリキャッシュ** | Service Worker Cache | HTML/CSS/JS、マップ画像 | **半永久的** |

**特徴**:
- 明示的に削除しない限り、基本的に永続的に保存されます
- ブラウザを閉じても、PCを再起動しても消えません
- オフライン状態でも利用可能です

### データが削除される条件

| 操作 | ユーザーデータ | アプリキャッシュ |
|------|--------------|----------------|
| アプリ内の「すべてのデータを削除」ボタン | ✅ 削除 | ❌ 残る |
| ブラウザの「閲覧履歴・Cookie・キャッシュを削除」 | ✅ 削除 | ✅ 削除 |
| ブラウザの「特定サイトのデータを削除」 | ✅ 削除 | ✅ 削除 |
| ブラウザのアンインストール | ✅ 削除 | ✅ 削除 |
| プライベートモード（シークレットモード） | ⚠️ セッション終了時に削除 | ⚠️ セッション終了時に削除 |

### ブラウザによる自動削除

以下の場合、ブラウザが自動的にデータを削除する**可能性**があります：

1. **ストレージ容量の逼迫**
   - デバイスのストレージ容量が極端に少なくなった場合
   - LocalStorageの容量制限（約5~10MB）を超えた場合
   - 最近使用されていないサイトのデータが優先的に削除される

2. **長期間未使用**
   - 一部のモバイルブラウザでは、数ヶ月以上アクセスしていないサイトのデータを自動削除することがあります

### データを失わないための推奨事項

重要なデータを保護するために、以下の対策を推奨します：

1. **定期的にバックアップ**
   - 「設定」→「コードで共有」でデータをテキスト形式でコピー
   - メモ帳やクラウドストレージに保存しておく

2. **PC・スマホ間で同期**
   - 「QRコードで共有」や「URLで共有」で複数デバイスにデータを分散保存
   - 一方のデバイスでデータが消えても、もう一方から復元可能

3. **ブラウザの設定確認**
   - ブラウザの「閲覧履歴を終了時に削除」設定をOFFにする
   - 特定のサイトのデータを保護する設定（Cookie許可など）を有効にする

## バージョン管理

このプロジェクトは [Semantic Versioning](https://semver.org/lang/ja/) を採用しています。

### バージョン形式: `v0.MINOR.PATCH`

- **メジャーバージョン (0.x.x)**: 正式リリース前のため0固定
- **マイナーバージョン (x.Y.x)**: 新機能追加時にインクリメント
- **パッチバージョン (x.x.Z)**: バグ修正・小規模改善時にインクリメント

※ バグ修正や細かなアップデートの場合も、右端（PATCH）の数字を増やします。

### 変更時に更新するファイル

1. `app/index.html` - アプリ情報のバージョン表示
2. `README.md` - バージョン履歴に追記
3. `app/sw.js`, `app/index.html` - キャッシュバージョン（`?v=XX`）
4. GitHub - ここまで更新したら **必ずpush**（端末のキャッシュ更新と手順の混乱防止）

### コミットメッセージのルール

- **形式**: `vX.Y.Z: 変更内容の概要` または `変更内容の概要 vX.Y.Z`
- **例**: `v0.6.0: コンパクトモード機能の追加` / `ドラッグ並び替えのバグ修正とバージョン更新 v0.5.6`
- バージョン番号を先頭または末尾に必ず含める
